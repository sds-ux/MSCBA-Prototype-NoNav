Processus serveur : Verifiez l'organisme avec les donné de la CESD


Check if BN already registered

When a user submits a business number (BN), a backend process performs the following test and redirects the flow accordingly.

  1. Tests if the BN is already registered with this program. If it is, generate a work item of type: CRA BN already registered.
  2. If there is a work item, set the status of the confirm organization step to: pending.

View error message

Confirm with another program

The next step is to check if the BN is registered with another MSCBA program. If it is, compare the data submitted by the user with the data from the other program using the following rule:

  1. The business number must be a perfect match.
  2. The legal name must be a perfect match
  3. The postal code must be a perfect match
  4. Attempt to match the country, province/state, city, and address lines 1 and 2 if data is available.
  5. Confirm the organization when all the data match
  6. When there is a mismatch, prevent confirmation and generate a work item of type: CRA BN - Information Validation

If we can confirm the organization, direct the user to the registration summary page where they can find the next incomplete step in the registration process.

Organization confirmed


If we cannot confirm the organization with another program, direct the user to the owner consent process. We do not want to generate a work item at this point while there is a chance to confirm the organization using CRA PAYDAC data.
