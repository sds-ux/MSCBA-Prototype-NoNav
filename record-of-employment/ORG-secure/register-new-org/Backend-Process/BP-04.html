Backend process: Confirm the organization with ESDC data


  1. The system verifies if the BN is already registered with the program

    2. If yes, the system:

    1. generates a work item of type “CRA BN already registered”
    2. bsets the status of the “Confirm your organization” step in the Registration summary page to “Pending”
    3. directs the user to an error message

    Fail: View error message

  2. The system verifies if the BN is registered with another MSCBA program

    3. This verification can only be done with programs that meet the same level of assurance (LOA) or higher.

    If yes, the system compares the following data submitted by the user with the data from the other program:

    1. business number
    2. organization legal name
    3. postal/zip code
        4. Note: The ESDC Identity Management Policy requires the entire address to match in order to confirm the organization. Currently, MSCBA only matches the postal code. To evaluate the fail/success rate if MSCBA was to include the other address fields in the validation process, the system attempts to match the country, province/state, city, and address lines 1 and 2 (if the other program has similar data collected) and records the hits and misses.

    The system confirms the organization only when all three data elements are a perfect match. When there is a mismatch, the system fails to confirm the organization and generates a work item of type “CRA BN - Information Validation”.

  3. The system redirects the user

    1. If the organization is confirmed with another program’s data, the system redirects the user to the Registration Summary page where they can find the next incomplete step in the registration process.

      2. Organization confirmed

    2. If the CRA business is confirmed, the system directs the user to the manage CRA businesses page where they can see that the CRA business in now active

      3. CRA business confirmed


    3. If the system cannot confirm the organization with another program’s data, the system directs the user to the business owner consent process where it will attempt to confirm the organization using CRA PAYDAC data (no work item generated at this point).

      4. Business owner consent - New organization

      Business owner consent - CRA business
