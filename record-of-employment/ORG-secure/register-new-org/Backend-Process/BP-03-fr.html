Processus en arrière-plan: Confirmer l’organisme avec les données de l’ARC


1 - Le système vérifie si le NE a l'extension RP

La base de données PAYDAC de l'ARC ne contient que les numéros d'entreprise (NE) avec l'extension RP. Si le NE a une extension différente, le Centre d'enregistrement des entreprises (BRC) doit manuellement confirmer l'organisme (cela est possible car le propriétaire d’entreprise a fourni son consentement à ce qu’EDSC confirme l’existence de son organisme auprès de l’ARC).

  • Le système vérifie si le NE a l'extension RP. Si ce n'est pas le cas, le système génère un élément de travail intitulé « Requiert autorisation manuelle ».

    • 2 - Le système confirme automatiquement l'organisme en utilisant la base de données PAYDAC de l’ARC

    Une fois que le propriétaire de l'entreprise a donné son consentement, le système utilise le numéro d'entreprise (BN15) pour extraire les données de l’ARC de la base de données PAYDAC. Si le système ne trouve pas le NE dans PAYDAC, le système génère un élément de travail intitulé « NE ARC non trouvé dans Paydac ».

    Si le système trouve le NE dans PAYDAC, il compare les données de PAYDAC avec les données de l'organisme soumises par l'utilisateur en appliquant les règles suivantes :

    1. Le numéro d'entreprise doit correspondre parfaitement
    2. Le nom légal doit correspondre parfaitement
    3. Le code postal/zip, le pays, la province/l'état et la ville doivent correspondre parfaitement
    4. Le système vérifie que le statut de l'organisme auprès de l'ARC est identifié comme étant « actif ». S'il ne l'est pas, le système génère un élément de travail intitulé « Statut ARC invalide ».
    5. Le système confirme l'organisme lorsque toutes les données correspondent
    6. Lorsque les données ne correspondent pas, le système empêche la confirmation de l’organisme et génère un élément de travail intitulé « NE ARC - Validation de l'information ».

    La politique de gestion de l'identité d'EDSC stipule que nous devons comparer toutes les composantes de l'adresse (et non seulement le code postal). Les règles de confidentialité, quant à elles, exigent que toutes les données que nous recueillons soient utilisées à des fins commerciales. Pour satisfaire à ces deux règles, le système collecte et compare les lignes d'adresse 1 et 2 en appliquant les règles suivantes :

    1. Le système confirme toujours l'organisme, quel que soit le résultat des lignes d'adresse 1 et
    2. Le système comptabilise les cas où les lignes d'adresse 1 et 2 correspondent et ceux où elles ne correspondent pas, afin que nous puissions déterminer si le nombre de cas qui ne correspondent pas se situe dans des limites acceptables.

    3 - Le système attribue automatiquement le rôle d’agent principal à l'utilisateur.

    1. L'utilisateur a confirmé son identité
    2. L'utilisateur a confirmé son statut en tant que propriétaire d'entreprise

    Une fois l'organisme confirmé, le système dirige l'utilisateur vers la dernière étape incomplète du processus d'inscription :

    1. Pour RE Web et le FPTAVE, l'utilisateur doit accepter l'Entente en ligne.
    2. Les SELSC n'ont pas d'entente en ligne; l’utilisateur est dirigé vers la page de l'organisme actif où il peut accéder au programme.

    Poursuivre


    Lorsqu'un élément de travail a été généré, le système dirige l'utilisateur vers le message d'erreur générique.
